В конфигурационном файле содержатся токен для загрузки файла на сервер, токен для просмотра файлов, что находятся на сервере, и их скачивания, порт,
а также имя хоста.
После первого запуска приложение автоматически создаёт шаблонный файл конфигурации config.json в папке "report-server" в домашней директории пользователя.
Он содержит значения по умолчанию (порт - 8080, имя - "localhost", токены - пустые строки), однако не может быть использован сервером в изначальном
виде. Чтобы сервер мог работать, в файл необходимо ввести токены. Конфигурационный файл рекомендуется создать самостоятельно до первого запуска приложения,
чтобы избежать ошибки.
